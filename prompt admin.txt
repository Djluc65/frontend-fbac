Agis comme un architecte logiciel senior et un développeur full-stack spécialisé en React, TypeScript, Node.js, Express, MongoDB, Redux Toolkit, RTK Query et systèmes CMS.

Je développe le site de la Fondation Bien Aimé Cassis.

Le site contient déjà :

- un frontend public en React ;
- un espace utilisateur ;
- un espace administrateur ;
- un backend Node.js / Express ;
- une base MongoDB ;
- une authentification JWT avec access token et refresh token ;
- des rôles administrateur et utilisateur.

Je veux maintenant transformer l’espace administrateur en véritable CMS afin de pouvoir modifier tous les contenus du site public sans passer par le code source.

==================================================
1. OBJECTIF PRINCIPAL
==================================================

Créer dans l’espace administrateur une section appelée :

“Gestion du contenu”

Cette section doit permettre aux administrateurs autorisés de modifier :

- les textes ;
- les titres ;
- les sous-titres ;
- les descriptions ;
- les images ;
- les boutons ;
- les liens ;
- les statistiques ;
- les programmes ;
- les campagnes ;
- les témoignages ;
- les partenaires ;
- les actualités ;
- les événements ;
- les coordonnées ;
- les réseaux sociaux ;
- les informations du footer ;
- les métadonnées SEO ;
- les traductions ;
- les paramètres généraux du site.

L’objectif est qu’un administrateur non développeur puisse gérer le contenu complet du site public depuis l’interface d’administration.

==================================================
2. CONTRAINTE IMPORTANTE
==================================================

Ne pas stocker les contenus importants directement dans les composants React.

Le frontend public doit récupérer les contenus depuis l’API backend.

Les composants React doivent afficher dynamiquement les données reçues depuis MongoDB.

Prévoir des données de secours uniquement si l’API est temporairement indisponible.
