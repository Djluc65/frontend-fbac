==================================================
11. REJET D’UN PAIEMENT
==================================================

Exiger une raison obligatoire.

Raisons possibles :

- référence introuvable ;
- preuve illisible ;
- montant incorrect ;
- destinataire incorrect ;
- devise incorrecte ;
- preuve déjà utilisée ;
- paiement non reçu ;
- suspicion de fraude ;
- autre.

Après rejet :

- Donation.status = REJECTED ou PENDING selon la logique métier ;
- Proof.status = REJECTED ;
- enregistrer la raison ;
- notifier l’utilisateur ;
- autoriser une nouvelle preuve si nécessaire ;
- ajouter un AuditLog.
